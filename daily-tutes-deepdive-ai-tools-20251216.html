2. Multi-Agent Systems Explained

What Are Multi-Agent Systems?

A multi-agent system is an architecture where multiple autonomous AI agents work together to accomplish complex goals that would be difficult or impossible for a single agent to achieve alone.

Key Characteristics

Core Principles of Multi-Agent Systems

  1. Autonomy: Each agent can make decisions independently
  2. Specialization: Agents focus on specific domains or tasks
  3. Communication: Agents share information and coordinate actions
  4. Collaboration: Agents work toward a common goal
  5. Emergence: Complex behaviors arise from simple interactions

Multi-Agent vs Multi-Model: Critical Distinction

When to Use Multi-Agent Systems

  • Tasks have distinct phases requiring different expertise
  • You need parallel processing of independent subtasks
  • Different parts of the workflow have conflicting requirements
  • You want to scale individual agents independently

3. CrewAI: High-Level Agent Orchestration

What is CrewAI?

CrewAI is a framework for building multi-agent systems using a role-based approach. You define agents by their role, goal, and backstory, then let them collaborate autonomously to accomplish tasks.

ğŸ¯ CrewAI Philosophy

"Define what your agents should do and who they are, not how they should do it. Let the framework handle orchestration."

Core Concepts

1. Agents

Each agent has:

  • Role: Their job title (e.g., "Research Analyst")
  • Goal: What they're trying to achieve
  • Backstory: Context that shapes their behavior
  • Tools: Resources they can use
  • LLM: The language model powering them
# CrewAI Agent Example from crewai import Agent from langchain.llms import OpenAI research_agent = Agent( role="Senior Research Analyst", goal="Discover cutting-edge developments in AI", backstory="""You're a seasoned researcher with a PhD in Computer Science. You have 15 years of experience tracking AI trends and can identify breakthrough technologies before they become mainstream.""", tools=[web_search_tool, arxiv_tool], llm=OpenAI(model="gpt-4"), verbose=True, allow_delegation=False )

2. Tasks

Tasks define what needs to be done:

from crewai import Task research_task = Task( description="""Research the latest developments in multimodal AI models. Focus on models released in the last 3 months. Provide a summary of top 5 breakthroughs with links to papers.""", agent=research_agent, expected_output="A markdown report with 5 breakthroughs, each with description and paper link" )

3. Crews

Crews coordinate multiple agents:

from crewai import Crew, Process # Define more agents analysis_agent = Agent( role="AI Technology Analyst", goal="Evaluate the practical implications of AI research", backstory="You translate academic research into business value...", tools=[calculator_tool, database_tool], llm=OpenAI(model="gpt-4") ) writing_agent = Agent( role="Technical Writer", goal="Create clear, engaging technical content", backstory="You're an award-winning tech journalist...", tools=[grammar_checker], llm=OpenAI(model="gpt-4") ) # Create tasks for each analysis_task = Task( description="Analyze the research findings and identify business opportunities", agent=analysis_agent ) writing_task = Task( description="Write a 1000-word blog post about the findings", agent=writing_agent ) # Assemble the crew crew = Crew( agents=[research_agent, analysis_agent, writing_agent], tasks=[research_task, analysis_task, writing_task], process=Process.sequential, # or Process.hierarchical verbose=True ) # Execute result = crew.kickoff() print(result)

Process Type How It Works Best For

Hierarchical Manager agent delegates tasks to worker agents Complex coordination, dynamic task allocation

ğŸ’ª CrewAI Strengths:
  • High-level abstraction: Focus on "what" not "how"
  • Natural delegation: Agents can delegate subtasks to each other
  • Built-in collaboration: Automatic communication between agents
  • Quick prototyping: Get multi-agent systems running in minutes
  • Flexible tools: Easy to add custom tools to agents

4. LangGraph: Graph-Based Workflows

What is LangGraph?

LangGraph is a framework for building stateful, graph-based agent workflows. Unlike CrewAI's high-level approach, LangGraph gives you explicit control over the flow of execution through nodes and edges.

ğŸ¯ LangGraph Philosophy

"Model your agent system as a graph where nodes are actions and edges are transitions. You control exactly when and how agents interact."

Core Concepts

1. State

The shared data structure that flows through your graph:

from typing import TypedDict, Annotated import operator class State(TypedDict): messages: Annotated[list, operator.add] # Append-only list query: str documents: list answer: str next_step: str

2. Nodes

Functions that operate on state:

def research_node(state: State) -> State: """Fetch relevant documents.""" query = state["query"] documents = search_web(query) return { **state, "documents": documents, "messages": state["messages"] + [ {"role": "system", "content": f"Found {len(documents)} documents"} ] } def analysis_node(state: State) -> State: """Analyze documents and generate answer.""" docs = state["documents"] query = state["query"] # Use LLM to analyze answer = llm.invoke(f"Query: {query}\n\nDocs: {docs}") return { **state, "answer": answer, "messages": state["messages"] + [ {"role": "assistant", "content": answer} ] }

3. Edges

Define transitions between nodes:

from langgraph.graph import StateGraph, END # Create graph workflow = StateGraph(State) # Add nodes workflow.add_node("research", research_node) workflow.add_node("analysis", analysis_node) workflow.add_node("quality_check", quality_check_node) # Add edges workflow.add_edge("research", "analysis") # Conditional edge def should_revise(state: State) -> str: """Decide if answer needs revision.""" if quality_score(state["answer"]) < 0.7: return "research" # Go back and research more else: return END # Finish workflow.add_conditional_edges( "quality_check", should_revise, { "research": "research", END: END } ) # Set entry point workflow.set_entry_point("research") # Compile app = workflow.compile()

Key Features

ğŸ’ª LangGraph Strengths:
  • Explicit control: You define exact flow logic
  • State management: Clear state object throughout execution
  • Cycles supported: Can loop back to previous nodes
  • Conditional routing: Dynamic decisions about next steps
  • Visualizable: Graph structure is easy to understand and debug
  • Streaming support: Can stream partial results

  • Debugging and observability are priorities

5. CrewAI vs LangGraph: Complete Comparison

The Big Picture

ğŸ­ The Analogy

CrewAI: Like hiring a team with clear job descriptions and letting them figure out how to collaborate. You're the CEO setting direction.

LangGraph: Like creating a detailed flowchart of exactly who does what, when, and under what conditions. You're the process engineer.

Detailed Comparison

Aspect CrewAI LangGraph
Abstraction Level High - declarative, role-based Low - explicit, graph-based
Learning Curve Easy - intuitive agent metaphor Moderate - requires graph thinking
Control Framework decides execution flow You control exact execution flow
Best For Complex multi-agent collaboration Precise workflow control
Loops/Cycles Limited - sequential or hierarchical Full support - explicit cycles

Debugging Can be opaque Clear graph visualization

Typical Use Case Content creation, research pipelines RAG systems, decision trees

Side-by-Side Code Comparison

Same Task: Research + Analysis + Writing

CrewAI Approach:

# CrewAI: ~30 lines research_agent = Agent( role="Researcher", goal="Find information about AI trends", tools=[web_search] ) analyst_agent = Agent( role="Analyst", goal="Analyze research findings", tools=[calculator] ) writer_agent = Agent( role="Writer", goal="Write report", tools=[grammar_check] ) crew = Crew( agents=[research_agent, analyst_agent, writer_agent], tasks=[research_task, analysis_task, writing_task], process=Process.sequential ) result = crew.kickoff() # Done!

LangGraph Approach:

# LangGraph: ~60 lines class State(TypedDict): query: str research: str analysis: str report: str def research_node(state): research = web_search(state["query"]) return {"research": research} def analysis_node(state): analysis = llm.analyze(state["research"]) return {"analysis": analysis} def writing_node(state): report = llm.write_report( state["research"], state["analysis"] ) return {"report": report} workflow = StateGraph(State) workflow.add_node("research", research_node) workflow.add_node("analysis", analysis_node) workflow.add_node("writing", writing_node) workflow.add_edge("research", "analysis") workflow.add_edge("analysis", "writing") workflow.add_edge("writing", END) workflow.set_entry_point("research") app = workflow.compile() result = app.invoke({"query": "AI trends"})
